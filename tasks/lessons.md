# Lessons

最后更新：2026-06-08

## 项目协作规则

- 对非琐碎任务，先更新 `tasks/todo.md`，再执行实现。
- 多 agent 并行时，先维护 `tasks/agent_context.md` 和 `tasks/parallel_board.md`，避免上下文分叉。
- 子 agent 的业务歧义统一回到主会话确认，不让多个会话同时打扰用户。
- 成功标准必须以业务复核结果为准，不能只看脚本退出码或保存请求命中。
- 不要为了复用旧链路把业务负担转移给用户；例如要求业务改图片文件名、整理目录、压缩包匹配，会抵消自动化价值。优先按业务真实操作方式设计输入。
- 单计划多渠道不能只用一个“推送内容”字段替代所有渠道文案；组合渠道必须允许 `短信内容` 和 `发送内容` 分别配置，`推送内容` 只能作为空字段兜底。
- 看到“已有逻辑要保留”时必须用测试或代码路径确认实际是否命中，例如组合渠道创建链接不能只确认映射存在，还要验证 key 匹配方式。
- 真实网页自测必须使用独立测试浏览器或明确的 CDP 目标，不要直接操作用户当前 Chrome 窗口，避免干扰已有业务标签页。
- 自动化结果不能只看任务最终 success；字段级日志里出现 `⚠️`、`回读为0`、`疑似重叠` 时要作为复核风险记录，必要时拆成后续修复任务。
- `/simple` 回归样本必须长期复用 `docs/simple_text_samples.md`。新增渠道组合或新增字段时，在这份文件上增补，不另起临时样本体系。
- 测试中遇到的失败样本、失败原因、真实链接、修复方式要写入 `tasks/todo.md` 的 Review；可复用的方法论再沉淀到本文件。
- 样本数据要用真实回读通过的值。已失败的商品编码、券规则 ID、图片素材、门店文件不能继续作为默认回归数据，否则下次会重复踩同一类问题。
- 第2步目标人群回归不能只看主页面能打开；必须确认 iframe 是 `cdp.dslyy.com` 正常页面，不是 `chrome-error://chromewebdata/`。
- 任务状态为 success 但字段清单有 fail 时不能视为完全通过；应修复字段复核标记或解释为明确的待复核项。
- 浏览器插件/侧边栏如果会缓存上一次复核数据，复核前必须校验“期望计划名称”和“当前页面计划名称”一致；不一致时要阻断对比并提示重新载入对应计划数据。
- Chrome 扩展“重新加载”不会清空 `chrome.storage.local`。如果旧缓存可能造成业务误判，应在插件启动或不匹配时主动清理存储，而不是要求用户理解浏览器扩展缓存机制。
- Chrome 扩展重新加载后，业务页面里旧 content script 可能继续运行但扩展上下文已失效。所有 `chrome.runtime.sendMessage` 这类调用都要包同步 `try/catch`，并在失败时断开 observer/timer。
- 带 `--skip-step2` 的真实回归不能表述为完整通过；涉及目标人群的需求必须单独标注“第2步未验证/待内网回归”。
- 字段填充不能只看脚本日志里的“已填充”；必须用业务页面可见值或保存后复核页确认。尤其是 Vue/Element 表单，直接改 DOM value 可能不进入表单模型。
- 表格型字段不要用宽泛 `div` 搜索；必须锁定真实行/列关系，例如“参数名称=活动介绍”同一行右侧“参数详情”，否则容易写到模板值、禁用控件或隐藏副本。
- Chrome 插件复核不能只按“字段名旁边就是值”的普通表单结构读取。智能电话这类自定义参数页要支持“参数名称=字段名 / 参数详情=字段值”的行列关系；否则当前页明明可见也会误判为待复核。
- Chrome 插件读取业务系统时要考虑 iframe：第2步目标人群常在 `cdp.dslyy.com` 内嵌页，侧边栏不能只读主 frame。弹窗/iframe 已打开时应合并所有可访问 frame 的字段；未打开时提示业务打开，不默认自动点击弹窗入口以免改变页面状态。
- 发布包升级/删除旧包前，必须确认 8790 上的旧 uvicorn 进程已停止。旧服务即使包目录被移到废纸篓，仍会按旧 `DATA_DIR` 写文件，造成 `No such file or directory`。启动器后续要能识别“端口已开但 cwd 是旧包/废纸篓”的状态，不能只因为端口存在就直接打开页面。
- 自动更新不能只替换 `app/`；根目录 `start.bat/start.command` 也是运行逻辑的一部分。涉及启动策略修复时，发布包、自动更新脚本和后端自愈都要同步外层启动器，否则业务电脑可能拿到新 app 但仍用旧启动逻辑。
- 业务素材上传不能只按扩展名或文件头判断图片可用。会员通素材接口可能拒绝部分 PNG 变体、透明通道或异常编码；UI 上传入口应先用图片库真实解码并规范化为 RGB JPEG，再交给 Playwright 上传。
- `/simple` 草稿功能的业务语义是“保存当前页面工作现场”，不是只保存文本字段。保存草稿应包含文本和图片顺序，进入页面自动恢复；清空草稿必须同时清空当前页面和持久化数据，不能只删 localStorage 后让输入框残留。

- 不同渠道的 cdp.dslyy.com 分群页面结构可能完全不同（如智能电话不含商品编码/券规则/主消费营运区）。第2步严格校验必须先检测页面是否实际包含目标字段，不存在时降级为警告而非失败。不能假设所有渠道的分群页面都有相同的字段集合。
- Windows 业务端反馈“无访问权限”时要先看浏览器错误类型：若地址是 `127.0.0.1:8790/simple` 且提示“拒绝了连接请求/ERR_CONNECTION_REFUSED”，这不是账号权限问题，而是本地 uvicorn 服务未启动或启动后崩溃；优先收集 `data\logs\ui_server.log` 和启动命令窗口输出。
- Windows `.bat` 在括号代码块里不能随便 `echo` 未转义的 `)`，例如 `echo ... (first run ...)` 会触发 `was unexpected at this time` 并中断启动；启动器提示文案优先避免括号，或显式转义。
- Windows 自动更新不能在 `start.bat` 正在运行时直接覆盖根目录 `start.bat`；批处理边执行边被替换可能出现空指令、跳行或异常退出。Windows 启动器更新应写入 `start.bat.pending`，由下一次外层启动器退出自身后再安全替换并重启。
- Windows 离线 wheelhouse 要显式覆盖带环境标记的依赖。`click` 在 Windows 下需要 `colorama`，即使 Mac 构建机本地测试不需要，也必须打进 Windows 发布包，否则离线 pip 会报 `No matching distribution found for colorama`。
- 执行员工级联选择不能因字段已有任意默认值且无校验报错就放行；必须回读命中目标文本或目标核心词。否则模板默认值（如全国/大量门店标签）会掩盖实际未选择目标区域。
