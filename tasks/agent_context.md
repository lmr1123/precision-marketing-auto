# 精准营销自动化多 Agent 共享上下文

最后更新：2026-05-22

## 项目定位

项目名称：精准营销自动化。

当前形态：业务版 FastAPI UI + Playwright 执行引擎。业务上传任务文件后，系统接管已登录 Chrome/CDP，在精准营销平台自动创建计划。

本地目录：`/Users/liminrong/precision-marketing-auto`

核心文件：

- `ui_app/server.py`：FastAPI 服务、上传解析、任务队列、内嵌前端页面。
- `precision-auto-playwright-batch.py`：Playwright 执行引擎，负责第1步/第2步/第3步填充、保存、日志输出。
- `tasks/todo.md`：当前任务计划与 review。
- `tasks/parallel_board.md`：并行 agent 看板。
- `tasks/lessons.md`：用户纠正与经验沉淀。

## 业务目标

业务试运行后发现，当前“上传 Excel/CSV -> 任务列表 -> 补素材 -> 开始执行 -> 看日志/重试”的流程仍然容易中断。

升级目标：

1. 业务只上传要配置的文本和图片，不再要求维护 Excel。
2. 上传后系统按每条计划旁边上传的图片顺序自动填写，不要求业务改图片文件名、整理目录或压缩包匹配。
3. 创建后返回每一条计划的创建结果、创建成功链接、复核结果。
4. 复核结果是第一优先级，成功不能只靠日志或弱成功信号判断。
5. 字段补齐不再逐字段手工试错，优先评估用 Chrome/Kimi WebBridge/LLM 辅助观察页面、学习人工操作、生成稳定自动化实现。

## 当前架构事实

- UI 已支持上传 CSV/XLSX/ZIP，上传后拆成单计划任务。
- UI 目前默认不自动执行，需要点击“开始执行”。
- 上传解析会把中文表头映射为内部英文字段，并做部分默认值补齐。
- Playwright 引擎已有字段级结果清单，但主要以日志输出，UI 通过正则解析进度和链接。
- CDP 接管同一浏览器上下文时应串行执行，避免互相抢页。
- 历史失败主要集中在：第2步 iframe 不稳定、执行员工区域选择残留、保存后 about:blank/弱成功、图片上传和小程序配置、CDP 连接失败。

## 关键原则

- 不把“脚本跑完”定义为业务成功。成功必须包含：创建成功、拿到可复核链接、关键字段复核通过。
- 不让业务读日志定位问题。UI/结果文件必须给出可读的逐条原因。
- 不在旧 Excel 流程上继续叠加复杂补丁。新流程应以文本+图片上传为主，Excel 兼容作为过渡。
- AI/浏览器新能力优先用于开发辅助、页面观察、字段补齐和复核，不直接作为不可控的业务运行时主链路。
- Playwright 运行时仍应尽量确定性、可复现、可审计。

## 并行开发规则

- 所有 agent 开始前必须阅读本文件和 `tasks/parallel_board.md`。
- 子 agent 遇到业务歧义，不直接假设为最终结论；把问题回报主会话。
- 子 agent 不得重置、删除或覆盖他人未提交改动。
- 子 agent 应只在明确授权的文件范围内修改。只读盘点 agent 不改业务代码。
- 主会话负责合并、验收和最终对用户汇报。

## 推荐第一阶段范围

第一阶段目标是跑通最小闭环，不追求一次性覆盖全部字段：

1. 新建一个简洁操作页面，和现有复杂任务中心分离。
2. 将文本解析为内部标准计划结构。
3. 页面支持批量上传 `.txt`，也支持逐条粘贴文本。
4. 每条文本计划旁边提供单张/多张图片上传控件，按业务选择顺序写入系统。
5. 点击执行后自动创建任务并运行，页面展示每行结果、链接和直接错误说明。
6. 结构化结果仍应输出，便于后续页面展示和排错。
7. 业务系统内的最终复核侧边栏插件作为后续协同方向；当前项目先保留结构化结果和基础复核能力接口。

## 已确认决策

- 第一阶段只支持强约束中文键值文本模板。
- 同时支持 `.txt` 文件上传和页面粘贴框。
- 不采用“业务改图片文件名 + zip + 计划图片ID 匹配”的主流程。
- 图片按每条计划上传控件中的选择顺序执行，贴近业务真实操作。
- 新建简洁操作页，业务主流程只保留：文本上传/粘贴、对应图片上传、点击执行、查看逐行结果。
- 简洁页先放在 `/simple` 试用，不替换旧首页。
- 图片顺序规则已确认：客户消息且有小程序字段时第1张为小程序封面，第2张起为内容图；朋友圈/社群按上传顺序作为内容图；短信忽略图片并提示。

## 当前待确认问题

- 每条文本计划最多允许上传几张图片；是否仍按朋友圈最多 9 张、小程序封面 1 张的规则自动区分。
- 是否需要接入云存储；3-5 人试运行阶段可以先本地存储。
- 未覆盖字段是否由用户录屏/现场操作一次，供 agent 学习页面行为。
