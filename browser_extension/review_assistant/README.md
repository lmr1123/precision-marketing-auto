# 精准营销复核助手

Chrome 扩展 MVP，用于在业务系统页面中对比 `/simple` 创建数据和当前页面可见字段。

## 使用方式

1. 打开 Chrome 扩展管理页：`chrome://extensions/`。
2. 开启“开发者模式”。
3. 点击“加载已解压的扩展程序”，选择本目录。
4. 在 `/simple` 执行结果里点击“复制复核数据”，或直接复制强约束计划文本。
5. 打开创建结果链接，进入精准营销业务页面。
6. 点击浏览器扩展图标，打开侧边栏。
7. 粘贴复核 JSON 或计划文本，点击“文本复核”。
8. 如需视觉辅助复核，在侧边栏填写“视觉服务地址”。服务地址由管理员提供，例如 `https://review.example.com` 或 `http://内网IP:8790`。
9. 如果仍有“待人工复核”字段，可以点击“截图复核”。插件会截取当前可见页面，调用统一视觉代理服务 `/api/review/vision`，由服务端读取 Ark Key。

注意：侧边栏不再自动恢复上一次复核数据，避免切换计划后误用旧数据。每次复核不同计划时，都重新粘贴对应复核数据或原始计划文本并载入；如果载入数据的计划名称和当前页面计划名称不一致，插件会清空旧数据并提示“数据不匹配”。

## 第一版边界

- 只读取当前页面 DOM 可见内容，不读取 cookie、localStorage、鉴权 header。
- 输入支持两种格式：`/simple` 复制出的复核 JSON，或强约束计划文本。
- 复核前会先校验计划名称；计划名称不一致时会清空已载入数据，不继续做字段对比，避免把旧计划数据误用于新页面。
- 视觉模型只在手动点击“截图复核”时调用；DOM 已经能判断的字段不会调用模型。
- Ark/Doubao 模型 Key 只保存在视觉代理服务端，插件只保存服务地址。
- 同一页面状态和同一批字段会缓存视觉结果，避免重复调用。
- 单页面最多调用 3 次视觉复核，超过后提示人工确认。
- 为了截图当前业务页面，扩展声明了 `<all_urls>` 主机权限；实际截图只在用户手动点击“视觉复核待复核项”时触发。
- 不自动修改业务系统字段。
- DOM 读不到的字段显示“待人工复核”，不误报通过。

## 后续预留

- 后续可把整页截图优化为字段区域截图，进一步降低费用和敏感信息外发范围。
- 视觉代理服务接口为 `POST /api/review/vision`，只从服务端环境变量 `ARK_API_KEY` 读取密钥；不要把密钥写入扩展代码。

## 稳定共享版部署

管理员在一台内网服务器或云服务器上部署本项目 UI 服务，并在 `.env.local` 配置：

```bash
ARK_API_KEY=你的 Ark Key
ARK_VISION_MODEL=doubao-seed-2-0-lite-260428
```

启动服务时监听可被同事访问的地址，例如：

```bash
.venv/bin/python -m uvicorn ui_app.server:app --host 0.0.0.0 --port 8790
```

业务同事只需要安装插件，并在侧边栏填写：

- 视觉服务地址：`http://服务器IP:8790` 或管理员提供的 HTTPS 地址
